Documentación

Animaciones morph
para modales

MorphCore transforma cualquier elemento en un modal animado con efecto morph. Escalable, configurable, compatible con ESM/UMD, y listo para SPAs.

GSAP powered UMD + ESM ~5KB min A11y ready
Características

¿Por qué MorphCore?

🎭

Efecto Morph

El elemento se transforma suavemente en el modal, creando una transición visual fluida y natural.

⚡

GSAP Powered

Usa GSAP para animaciones de alto rendimiento a 60fps con easings profesionales.

📦

UMD + ESM

Compatible con <script> tags, import/export ES Modules, y require() CommonJS.

🔧

Configurable

Defaults globales con configure(), overrides por modal con data-*, y overrides programáticos en open().

♿

Accesible

Focus trap, role="dialog", aria-modal, aria-labelledby, y cierre con Escape automáticos.

🔄

SPAs Ready

register() para modales dinámicos, destroy() para cleanup, sistema de eventos on/off.

Inicio rápido

Instalación y uso básico

  1. Incluir GSAP y MorphCore 
    <!-- CDN -->
<script src="https://cdn.jsdelivr.net/npm/gsap@3.14.2/dist/gsap.min.js"></script>
<script src="Morphcore.js"></script>

// O con ES Modules:
import MorphCore from './Morphcore.js';
  2. Crear el HTML del modal 
    <div class="modal"
     data-w="400"
     data-h="300"
     data-bg="#ffffff"
     data-radius="16px"
     data-pos="center">

  <span>Click para abrir</span>

  <div class="modal-content">
    <h2>Título</h2>
    <p>Contenido del modal</p>
    <button data-close>Cerrar</button>
  </div>
</div>
  3. Inicializar MorphCore 
    // Configuración opcional antes de init
MorphCore.configure({
  dur: 0.5,
  ease: 'expo.out',
  debug: true,
});

// Inicializar
MorphCore.init({
  onOpen:  (m) => console.log('Abierto:', m.id),
  onClose: (m) => console.log('Cerrado:', m.id),
});
Nota: El overlay es opcional por defecto. Los modales se abren sin overlay. Para habilitarlo, configura overlayAutoCreate: true y overlayColor: 'rgba(0,0,0,0.5)'.
Estructura

Anatomía de un modal

Cada modal tiene dos partes: el trigger (lo que se ve antes de abrir) y el contenido (lo que se muestra al abrir).

<div id="mi-modal" class="modal" data-w="500" data-h="400">

  <!-- TRIGGER: visible cuando el modal está cerrado -->
  <span>Soy el trigger</span>

  <!-- CONTENIDO: se clona y muestra al abrir -->
  <div class="modal-content">
    <h2>Título del modal</h2>
    <p>Contenido aquí...</p>
    <button data-close>Cerrar</button>
  </div>
</div>

El .modal-content está oculto por CSS (display: none). Al abrir, MorphCore lo clona dentro de un .morph-clone posicionado con position: fixed.

CSS mínimo requerido

/* Ocultar contenido cuando está cerrado */
.modal-content {
  display: none;
}

/* El clone animado */
.morph-clone {
  position: fixed;
  overflow: hidden;
  z-index: 100;
}

/* Mostrar contenido dentro del clone */
.morph-clone .modal-content {
  display: flex;
}

/* Ocultar trigger cuando está abierto */
.modal[data-estado="abierto"] > * {
  visibility: hidden;
  opacity: 0;
}
Referencia

API Completa

MorphCore.configure(options)

Actualiza los defaults globales. Se puede llamar antes o después de init(). Retorna el objeto de configuración actual.

MorphCore.configure({
  dur: 0.4,
  ease: 'power3.out',
  pad: 20,
  debug: true,
  overlayColor: 'rgba(0,0,0,0.6)',
  closeOnEscape: true,
});

MorphCore.init(options?)

Inicializa la librería. Registra todos los .modal existentes, crea el overlay si no existe, y configura los event listeners globales.

MorphCore.init({
  gsap: gsap,              // opcional si gsap está en window
  onOpen: (modal) => {},    // callback al abrir
  onClose: (modal) => {},   // callback al cerrar
  onComplete: (modal) => {},// callback al completar animación
  config: { dur: 0.5 },     // equivale a configure()
});

MorphCore.open(target, overrides?)

Abre un modal. target puede ser un elemento DOM, un ID (string), o un selector CSS.

// Por ID
MorphCore.open('mi-modal');

// Por selector
MorphCore.open('#mi-modal');
MorphCore.open('.modal.especial');

// Por referencia DOM
const el = document.getElementById('mi-modal');
MorphCore.open(el);

// Con overrides programáticos (ignoran data-*)
MorphCore.open('mi-modal', {
  w: 600,
  h: 400,
  bg: '#1a1a2e',
  radius: '24px',
});

MorphCore.close()

Cierra el modal superior del stack (el último abierto).

MorphCore.closeAll()

Cierra todos los modales abiertos, animando solo el último.

MorphCore.register(selector)

Registra modales dinámicamente (útil para SPAs). Acepta un selector CSS o un elemento DOM.

// Después de inyectar HTML dinámico
document.body.insertAdjacentHTML('beforeend', nuevoModalHTML);
MorphCore.register('#modal-nuevo');

MorphCore.isOpen(target)

Retorna true si el modal está actualmente abierto.

if (MorphCore.isOpen('mi-modal')) {
  console.log('El modal está abierto');
}

MorphCore.getState()

Retorna el estado interno de la librería.

const state = MorphCore.getState();
// {
//   initiated: true,
//   stackLength: 1,
//   scrollLocked: true,
//   config: { dur: 0.55, ... },
//   openModals: ['mi-modal']
// }

MorphCore.destroy()

Destruye la instancia: cierra todos los modales, remueve listeners, resetea configuración. Permite re-inicializar con init().

MorphCore.version

String con la versión actual: "2.0.0"

Layout

Posiciones

El atributo data-pos controla dónde se posiciona el modal al abrirse. Todas las posiciones respetan el data-pad como margen mínimo.

ValorDescripción
centerCentro del viewport (default). Se apila con offset si hay múltiples modales.
topCentrado horizontalmente, pegado arriba.
bottomCentrado horizontalmente, pegado abajo.
leftCentrado verticalmente, pegado a la izquierda.
rightCentrado verticalmente, pegado a la derecha.
top-leftEsquina superior izquierda.
top-rightEsquina superior derecha.
bottom-leftEsquina inferior izquierda.
bottom-rightEsquina inferior derecha.
originAl lado del trigger (derecha, o izquierda si no cabe).
origin-belowDebajo del trigger (o arriba si no cabe).
origin-aboveArriba del trigger (o debajo si no cabe).

Posición absoluta con data-pos-x / data-pos-y

Para posicionamiento preciso, usa data-pos-x y data-pos-y. Aceptan valores en px o %.

<!-- Posición absoluta en px -->
<div class="modal" data-pos-x="100" data-pos-y="200">

<!-- Posición en porcentaje -->
<div class="modal" data-pos-x="50%" data-pos-y="25%">
Extensibilidad

Sistema de Eventos

MorphCore v2 incluye un sistema de eventos pub/sub para extender la funcionalidad sin modificar el código fuente.

EventoArgumentoCuándo se emite
openmodal elementInmediatamente al iniciar la apertura
closemodal elementCuando la animación de cierre termina
completemodal elementCuando la animación de apertura termina
 
// Suscribirse a eventos
function onModalOpen(modal) {
  console.log('Se abrió:', modal.id);
  // Analytics, logging, etc.
}

MorphCore.on('open', onModalOpen);
MorphCore.on('close', (m) => console.log('Cerrado:', m.id));
MorphCore.on('complete', (m) => {
  // Cargar contenido async después de la animación
  fetchContent(m.id);
});

// Desuscribirse
MorphCore.off('open', onModalOpen);
Tip: Los callbacks pasados en init({ onOpen, onClose, onComplete }) se registran internamente como eventos, así que puedes agregar múltiples handlers con on().
Interacción

Triggers y acciones

Trigger automático (click en el .modal)

Todo elemento con clase .modal es clickeable como trigger. Al hacer click, se abre a sí mismo.

Trigger externo con data-morph-target

Cualquier elemento fuera del modal puede abrirlo usando data-morph-target:

<button data-morph-target="mi-modal">Abrir modal</button>

<div id="mi-modal" class="modal" ...>...</div>

Cerrar con data-close

Cualquier elemento dentro de .modal-content con data-close cierra el modal:

<button data-close>✕ Cerrar</button>

Abrir otro modal desde dentro

Usa data-morph-target dentro de .modal-content para encadenar modales:

<div class="modal-content">
  <h2>Modal 1</h2>
  <button data-morph-target="modal-2">Abrir Modal 2</button>
</div>
Avanzado

Uso en SPAs

En aplicaciones de página única (React, Vue, Svelte, etc.), los modales se crean y destruyen dinámicamente. MorphCore v2 soporta esto con register() y destroy().

Registrar modales dinámicos

// Después de montar un componente con un modal
onMounted(() => {
  MorphCore.register('#modal-dinamico');
});

// Al desmontar
onUnmounted(() => {
  MorphCore.destroy();
});

Abrir programáticamente

// Desde cualquier parte del código
function handleClick() {
  MorphCore.open('#modal-detalle', {
    w: 700,
    h: 500,
    bg: '#0f172a',
  });
}

Ciclo de vida completo

// 1. Configurar defaults
MorphCore.configure({ dur: 0.4, debug: false });

// 2. Inicializar
MorphCore.init();

// 3. Registrar modales dinámicos según se necesiten
MorphCore.register('.modal-lazy');

// 4. Escuchar eventos
MorphCore.on('open', trackAnalytics);

// 5. Consultar estado
console.log(MorphCore.getState());

// 6. Cleanup al desmontar la app
MorphCore.destroy();
Importante: Después de destroy(), puedes volver a llamar init() para re-inicializar. Esto es útil para hot-reload en desarrollo.

Cascada

Prioridad de configuración

MorphCore usa una cascada de 3 niveles para resolver opciones. El nivel más específico gana:

// Nivel 1: Defaults internos (más bajo)
{ dur: 0.55, ease: 'expo.out', pos: 'center', ... }

// Nivel 2: configure() — sobreescribe defaults
MorphCore.configure({ dur: 0.4 });

// Nivel 3: data-* en el HTML — sobreescribe configure()
<div data-dur="0.7">  <!-- este modal usa 0.7 -->

// Nivel 4: overrides en open() — máxima prioridad
MorphCore.open('#modal', { dur: 1.0 }); // usa 1.0
