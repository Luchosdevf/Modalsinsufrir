MorphModal

Librería vanilla JS + GSAP para animaciones morph de modales. El trigger se expande hasta convertirse en el modal — sin frameworks, sin dependencias extra.

→ Requiere GSAP 3. No requiere ScrollTrigger ni Flip. Solo el core de GSAP.

Instalación

CDN (recomendado)

html<!-- 1. GSAP primero -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/gsap.min.js"></script>

<!-- 2. MorphModal -->
<script src="https://cdn.jsdelivr.net/gh/TU_USUARIO/TU_REPO@latest/morphmodal.min.js"></script>

Estructura HTML

MorphModal necesita tres elementos clave: el overlay, el trigger y el contenido del modal.

html<!-- Overlay global (uno solo en todo el documento) -->
<div id="mo"></div>

<!-- Trigger + contenido anidado -->
<div class="modal-trigger"
     data-w="420"
     data-h="320"
     data-pos="center">

  Abrir modal

  <div class="modal-content">
    <h2>Título</h2>
    <p>Contenido del modal.</p>
    <button data-close>Cerrar</button>
  </div>

</div>
✓ .modal-content debe ser hijo directo del trigger. MorphModal lo clona y lo anima dentro del shell.
! El id="mo" del overlay es fijo. No lo cambies ni lo omitas — MorphModal lo busca por ese ID para animar el fondo.

CSS requerido

Este es el mínimo indispensable. Sin esto la librería no funciona correctamente.

css/* Overlay */
#mo {
  position: fixed;
  inset: 0;
  background: rgba(0, 0, 0, 0.5);
  opacity: 0;
  pointer-events: none;
  z-index: 999;
}

/* Ocultar el .modal-content original (solo se ve el clon animado) */
.modal-content {
  display: none;
}

/* El clon se posiciona desde JS — no tocar */
.morph-clone {
  position: fixed;
  z-index: 1000;
  overflow: hidden;
}

Estilo del contenido abierto

Para dar estilo al modal cuando está abierto, usá la clase .is-open que MorphModal agrega al clon del contenido:

css.modal-content.is-open {
  padding: 2rem;
  display: flex;
  flex-direction: column;
  gap: 1rem;
  /* cualquier estilo de layout/tipografía acá */
}

Init

Llamá a MorphModal.init() una sola vez, después de cargar GSAP.

jsMorphModal.init({
  gsap: gsap  // instancia de GSAP (requerido)
});
→ Si GSAP está en el scope global como window.gsap, podés llamar MorphModal.init() sin argumentos y lo detecta automáticamente.

Podés llamar init() antes del DOMContentLoaded — MorphModal espera internamente.

data-* API

Toda la configuración va en atributos del trigger. Sin JS adicional.

Atributo Tipo Default Descripción
data-w number 400 Ancho del modal en px
data-h number 400 Alto del modal en px
data-pos string center center · origin · custom
data-x number 50 % horizontal (solo con data-pos="custom")
data-y number 50 % vertical (solo con data-pos="custom")
data-dur number 0.9 Duración de la animación en segundos
data-radius string — Border-radius del modal abierto (ej: "16px")

Estilos por trigger

Cada trigger puede tener su propio look sin tocar la librería. La clave está en que MorphModal copia el background-color y el border del trigger al clon animado.

Ejemplo: tres triggers distintos

css/* Trigger rojo */
.trigger-danger {
  background-color: #7f1d1d;
  border-radius: 8px;
}

/* Trigger glassmorphism */
.trigger-glass {
  background-color: rgba(255,255,255,0.08);
  border: 1px solid rgba(255,255,255,0.12);
  border-radius: 16px;
  backdrop-filter: blur(14px);
}

/* Trigger pill */
.trigger-pill {
  background-color: #1e3a5f;
  border-radius: 999px;
  padding: 0.5rem 1.25rem;
}
html<div class="modal-trigger trigger-danger" data-w="400" data-h="300" data-radius="12px">
  ...
</div>

<div class="modal-trigger trigger-glass" data-w="480" data-h="360" data-radius="20px">
  ...
</div>
✓ El border-radius del trigger es el punto de partida de la animación. El data-radius es el border-radius del modal ya abierto. Cuanto más distintos, más dramático el morph.

Estilo del contenido por trigger

Para diferenciar el contenido de cada modal, dale una clase al .modal-content:

html<div class="modal-trigger trigger-danger" data-w="400" data-h="300">
  <div class="modal-content modal-danger">
    <h2>¡Atención!</h2>
    <button data-close>Entendido</button>
  </div>
</div>
css.modal-danger.is-open {
  padding: 2rem;
  border-top: 3px solid #ef4444;
}
.modal-danger.is-open h2 { color: #fca5a5; }

API pública

MétodoDescripción
MorphModal.init(config) Inicializa la librería. Registra todos los triggers existentes en el DOM.
MorphModal.open(el) Abre el modal del elemento trigger pasado como argumento.
MorphModal.close() Cierra el modal activo.

Abrir por código

jsconst trigger = document.querySelector("#mi-trigger");
MorphModal.open(trigger);

Cerrar por código

jsMorphModal.close();

Qué no tocar

!
No modificar con CSS:
  • .morph-clone — MorphModal controla su posición, tamaño y opacidad vía JS
  • #mo opacidad y pointer-events — los maneja la animación
  • .modal-content display — debe permanecer en none
✓
Podés modificar libremente:
  • Cualquier estilo visual del trigger (background, border, border-radius, padding, tipografía)
  • El background del overlay #mo (color y blur)
  • Todo el contenido dentro de .modal-content.is-open

Tips

El morph se ve mejor cuando...

El trigger y el modal tienen formas notablemente distintas. Un botón pill (border-radius: 999px) que se expande a un modal rectangular genera el efecto más llamativo.

data-pos="origin"

El modal se abre en la misma posición del trigger, sin moverse al centro. Útil para tarjetas que se expanden in-place.

Cerrar con Escape

Está habilitado automáticamente. No necesitás hacer nada.

Cerrar con overlay

El overlay #mo cierra el modal al hacer click. También automático.

Inputs dentro del modal

MorphModal preserva el estado de input, textarea y select entre aperturas. El usuario no pierde lo que escribió.

Múltiples triggers

Podés tener tantos triggers como quieras en la misma página. Solo puede estar abierto uno a la vez.
